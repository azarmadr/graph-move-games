Non-negotiable commitments:
- UI payload = full state every move.
- Spawning = deterministic DAG branching by enumerating spawn outcomes (no sampling).
- DAG acyclic by construction: new nodeIds are always increasing, and all edges added by `extend_path` go from the extended sink to newly created nodes (`from.nodeId < to.nodeId`).

Core spec:
- Board 4x4. Cells contain `{pos(r,c), tile}`; empties absent.
- Move valid iff `board_after_merges != board`.
- Merge scoring = standard 2048; no double-merge per tile.
- `extend_path(GamePathState, Direction) -> (GamePathState', GraphDelta)`:
  - If cursor sink is `Terminated`: no-op, `GraphDelta.isTerminated = true`, otherwise unchanged cursor/score.
  - If invalid move: no structural changes, empty delta; cursor/score unchanged; `GraphDelta.isTerminated` reflects current cursor termination status (false if still Active).
  - If valid:
    1) sink node kind `Sink{gameId,status=Active}` → `Regular`
    2) compute `B_merge` and `mergeScore`
    3) enumerate spawn outcomes from `B_merge` (empty-only placement)
    4) for each distinct spawn outcome, create a new `Sink` node with board = `B_i` and status Active iff further valid moves exist else Terminated
    5) add edges: `Move{direction}` and `Spawn{cells}` from old sink to each new sink
    6) score += mergeScore (spawn does not score)
    7) cursor chooses ONE deterministic branch: first outcome by stable ordering of spawned cells (sort by `(r,c)` then `tile`), and cursor becomes that chosen new sink
- Spawn enumeration rules:
  - outcomes are generated based on configuration (spawn count, value distribution, and any caps you implement)
  - outcomes must be deduplicated by exact multiset of spawned `(pos,tile)`
  - cap with `max_spawn_outcomes` (default 32) using deterministic stable ordering

Data model (JSON-ready):
- `Board { dim:(4,4), tiles: Vec<Cell> }`
- `Cell { pos:{r,c}, tile:u32 }`
- `Node { nodeId, board, kind }`
- `NodeKind = Source | Regular | Sink { gameId, status }`
- `Edge { edgeId, from, to, edgeType }`
- `edgeType = Move{direction} | Spawn{spawn:{cells:Vec<Cell>}}`
- `GameStatus = Active | Terminated`
- `GraphDelta = { nodesAdded: Vec<Node>, edgesAdded: Vec<Edge>, isTerminated: bool }`
  - `nodesAdded` and `edgesAdded` reflect only what was newly created
  - `isTerminated` is only whether the cursor’s game is terminated after this move

UI ↔ WASM contract (minimal):
- JS sends: `{ gameId, direction }`
- WASM returns:
  - `{ activeGameId, cursor:{gameId,sinkId,status,score}, activeBoard: Board, graph:{nodes,edges} }`
  - (No separate “incremental graph” is required; full state is returned every time.)

UI requirements:
- Board canvas: always render 4x4 grid + readable tile values + 2048-like colors; invalid moves show no board/score change.
